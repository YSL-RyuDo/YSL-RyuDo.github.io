UML Lesson 1: What is UML?

Temporary Draft

One-line Summary

UML is a standard modeling language for describing software structure and behavior with diagrams. It helps explain a system before implementation, or document a system after it has been built.

Why UML Helps

Game server and network systems can quickly become tangled with sessions, packets, matchmaking, rooms, and databases. UML makes those relationships easier to scan and gives the team a shared picture for discussing design.

UML is a Design Language, Not Code

UML does not run like C++ or C#. Instead, it visualizes class relationships, message flow between objects, and state transitions. Its job is to make design easier to understand, not to execute a program.

Common Diagram Types

  • Class Diagram: shows classes, fields, methods, inheritance, and dependencies.
  • Sequence Diagram: shows the order of messages between objects such as client, server, and database.
  • State Diagram: shows how an object changes state, such as a player session or matchmaking queue.
  • Use Case Diagram: shows what users can do with the system at a high level.

Game Server Example

For a login request, a Sequence Diagram can show the Client sending a request to the LoginServer, the LoginServer checking the DB, and the result returning to the Client.

Direction of These Notes

This series will focus less on memorizing every UML rule and more on using practical diagrams to explain portfolio projects and game server design decisions.
