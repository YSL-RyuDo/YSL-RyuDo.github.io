UML 1강: UML이란?

Temporary Draft

한 줄 요약

UML은 소프트웨어 구조와 동작을 그림으로 표현하기 위한 표준 모델링 언어입니다. 코드를 작성하기 전에 설계를 공유하거나, 이미 만든 시스템을 설명할 때 사용합니다.

UML이 필요한 이유

게임 서버나 네트워크 시스템은 세션, 패킷, 매칭, 룸, DB 같은 요소가 서로 얽히기 쉽습니다. UML을 사용하면 이런 관계를 글보다 빠르게 파악할 수 있고, 팀원과 같은 그림을 보며 설계를 이야기할 수 있습니다.

UML은 코드가 아니라 설계 언어

UML은 C++이나 C#처럼 실행되는 언어가 아닙니다. 대신 클래스의 관계, 객체 간 메시지 흐름, 상태 변화 같은 내용을 시각적으로 정리합니다. 즉, UML의 목적은 프로그램을 실행하는 것이 아니라 설계를 이해하기 쉽게 만드는 것입니다.

자주 쓰는 다이어그램

  • Class Diagram: 클래스, 필드, 메서드, 상속/의존 관계를 표현합니다.
  • Sequence Diagram: 클라이언트, 서버, DB 같은 객체들이 어떤 순서로 메시지를 주고받는지 표현합니다.
  • State Diagram: 캐릭터, 세션, 매칭 큐처럼 상태가 바뀌는 대상을 표현합니다.
  • Use Case Diagram: 사용자가 시스템으로 무엇을 할 수 있는지 큰 흐름을 표현합니다.

게임 서버 관점에서의 예시

예를 들어 로그인 요청을 설명할 때 Sequence Diagram을 사용하면, Client가 LoginServer에 요청을 보내고, LoginServer가 DB를 조회한 뒤, 결과를 Client에 반환하는 흐름을 한눈에 볼 수 있습니다.

이 노트의 방향

이 시리즈는 UML 문법을 전부 외우는 것보다, 포트폴리오와 게임 서버 설계를 설명하는 데 필요한 다이어그램을 실용적으로 정리하는 방향으로 작성할 예정입니다.

임시 메모: 나중에 실제 서버 예시 이미지와 PlantUML 또는 draw.io 기반 다이어그램을 추가할 예정입니다.
